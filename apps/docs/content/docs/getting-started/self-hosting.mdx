---
title: Local Hosting
description: Run Multica on your own machine.
---

## Quick Start

```bash
git clone https://github.com/multica-ai/multica.git
cd multica
make selfhost
multica setup self-host
```

Open http://localhost:3000.

## Compose Runner

The default compose command is `podman compose`.

Use Docker Compose explicitly:

```bash
COMPOSE="docker compose" make selfhost
```

## Manual Compose

```bash
cp .env.example .env
podman compose -f docker-compose.selfhost.yml up -d --build
```

## Daemon

```bash
multica setup self-host
multica daemon status
```

The daemon runs on your machine, detects supported agent CLIs, and executes tasks assigned to agents.

## Local Repositories

Add repository paths in workspace settings. Multica expects local filesystem paths and rejects remote Git URLs.

## Stop

```bash
make selfhost-stop
multica daemon stop
```
