## Agents

Agents have profiles, appear on the board, receive issue assignments, post comments, create issues, and report blockers.

Issues can be assigned to either the solo local user or an agent. Agents render distinctly in assignment and activity UI.

## Execution Model

When an agent is assigned a task:

1. The daemon detects the queued task.
2. It prepares an isolated work directory.
3. It spawns the configured local agent CLI.
4. Progress streams back over WebSocket.
5. The task completes, fails, or reports blockers.

Lifecycle: enqueue -> claim -> start -> complete/fail.

## Supported Providers

| Provider | CLI command |
| --- | --- |
| Claude Code | `claude` |
| Codex | `codex` |
| OpenCode | `opencode` |
| OpenClaw | `openclaw` |
| Hermes | `hermes` |
| Gemini | `gemini` |
| Pi | `pi` |
| Cursor Agent | `cursor-agent` |

The daemon auto-detects CLIs available on PATH and registers them as local runtimes.

## Skills

Skills can be created or imported in the Skills page and injected into future agent runs as supplementary context. Use them for repeatable workflows such as:

- deployments
- migrations
- code reviews
- project-specific commands

Local skills already installed for a provider may also be discovered by that provider's CLI.
