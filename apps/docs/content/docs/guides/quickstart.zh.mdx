---
title: Quickstart
description: Run Multica locally and assign your first task to an agent.
---

## 1. Start Multica

```bash
make selfhost
multica setup self-host
```

Open http://localhost:3000.

`multica setup self-host` configures the CLI for the local server, discovers workspaces, and starts the daemon. It does not authenticate.

## 2. Add a Local Repository

Open workspace settings and add a local repository path. Remote URLs are rejected; Multica expects paths on this machine or mounted volumes.

## 3. Verify Your Runtime

Go to **Settings -> Runtimes**. Your daemon should appear with detected agent CLIs.

Install at least one supported CLI if none are detected: `claude`, `codex`, `opencode`, `openclaw`, `hermes`, `gemini`, `pi`, or `cursor-agent`.

## 4. Create an Agent

Go to **Settings -> Agents**, create an agent, and select one of the detected local runtimes.

## 5. Assign a Task

Create an issue from the board or CLI:

```bash
multica issue create --title "Investigate failing test"
```

Assign it to your agent. The daemon will claim the task, run the selected CLI, and stream progress back to the issue.
