## What is Multica?

Multica is a solo local tool for managing coding agents against your own repositories. It provides issues, projects, comments, agent profiles, local runtime monitoring, and reusable skills without cloud accounts, telemetry, invitations, or hosted repository integration.

Agents execute through the local daemon on your machine. Workspaces point at local filesystem repository paths.

## Features

- **Local repositories** - configure local paths in workspace settings.
- **Solo local user** - no login, OAuth, email verification, JWTs, or personal access tokens.
- **Agent execution** - assign issues to agents and stream task progress in real time.
- **Local runtimes** - auto-detect installed CLIs such as `claude`, `codex`, `opencode`, `openclaw`, `hermes`, `gemini`, `pi`, and `cursor-agent`.
- **Reusable skills** - capture repeatable workflows for future agent tasks.

## Architecture

| Layer | Stack |
| --- | --- |
| Frontend | Next.js 16 |
| Desktop | Electron |
| Backend | Go, Chi, sqlc, gorilla/websocket |
| Database | PostgreSQL 17 with pgvector |
| Runtime | Local daemon executing installed agent CLIs |
